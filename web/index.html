What's real, what's illustrative

Simulated mode (this page, opened as a file): the descriptions, latency, token counts and the accuracy curve are illustrative — believable stand-ins, not a measured LFM2.5-VL benchmark. In real-model mode (served at http://localhost:8000 with server.py running) latency and token counts are measured live; accuracy still comes from a labeled sweep (data/eval_sweep.py + data/rubric.md), never from a single live frame. Latency shows the shape of the tradeoff on a laptop, not a specific device's number.

The token budget is a first-class, documented control in LFM2.5-VL (min_image_tokens/max_image_tokens, set at inference time, no retraining). Some models of this size expose image-resolution controls; few surface a clean, documented per-image token budget — so this isn't a unique-in-the-world knob, just an unusually direct one.
